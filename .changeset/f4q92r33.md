Implement asm-side bank + SRAM emission. Closes #96 — fills the
last asm gap that was deferred when Phase 2 closed, lets
`banks.gas` and `save.gas` ship as real examples.

## New directives

- `bank N` (sticky) — every subsequent statement emits into bank
  slot `N` (0-based) until the next `bank` directive or EOF.
  Labels inside a bank resolve to CPU addresses in the
  `$C000..$FEFF` window (where `mb = N` maps to the bank at
  runtime). Same shape as `org`.
- `sram_banks N` — header annotation declaring N battery-backed
  banks. SRAM occupies the **last** N bank slots per ISA §5; the
  host persists them via `int $21`.

Both accept a folded u8 hex literal (`bank $01`, `sram_banks $02`).

## Codegen

`assemble()` now tracks per-bank emit cursors via `Layout` +
`Emit` structs; the archive layout is base image followed by
each bank's 16 KB segment (zero-padded if the program didn't fill
the window, zero-filled for skipped bank indices). Header
`bank_count` = `max_bank + 1` when any bank was declared, else
`0`; `sram_bank_count` mirrors the latest `sram_banks` value.
Banked flag bit (header flags bit 0) is set whenever
`bank_count > 0`.

## Examples

- `examples/asm/banks/{main,bank0_greet,bank1_excite}.gas` —
  3-file demo of the recommended one-file-per-bank convention,
  with the bank includes at the **bottom** of the root so the
  sticky `bank N` directive doesn't bleed into the base image.
  Prints `Hi!\n` via cross-bank calls.
- `examples/asm/save.gas` — writes `"SAV"` into SRAM, flushes via
  `int $21`, prints `OK\n`. Produces a 16 KB `.sav` file.

## Spec

`docs/asm.md §5` rewritten from "deferred" to the full syntax
description (sticky directive, multi-file convention,
cross-bank call recipe). `bank N` removed from `§6` (Roadmap);
`bank_call` / `bank_jump` pseudo-instructions stay on the
roadmap for a future asm minor release.

## Out of scope (follow-up)

- `bank_call` / `bank_jump` sugar (user writes the 2-instruction
  sequence manually in v0.1).
- `org` in base image with a target in the bank-window range
  (`$C000..$FEFF`) — currently accepted as a bank-local offset
  via the same code path; left as user responsibility.
