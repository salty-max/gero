`gero run <file.gx>` now executes. The bare-VM target intercepts
two reserved `int N` syscalls before they reach the IVT:

- `int 0x10` prints the low byte of `r1` to stdout.
- `int 0x21` flushes the SRAM bytes to `<basename>.sav` next to
  the source `.gx`.

Exit codes per `cli.md` §3.3: `0` on `hlt`, `6` on unhandled
fault, `1` on host-level error (missing file / bad magic /
unsupported target), `2` on a `brk` breakpoint.
`--target=gtx-16` emits a "not implemented in this build"
message — the fantasy-console shim lives in a separate crate.

End-to-end smoke: a hand-rolled `.gx` running
`mov 'H', r1; int 0x10; mov 'I', r1; int 0x10; hlt`
through `gero run hello.gx` prints `HI` and exits `0`.
