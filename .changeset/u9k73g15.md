Add the asm parser scaffold + label-statement support per asm
spec §2.1. First slice of #33 — directives + instructions land
in follow-up PRs that each add one variant to `ast.Statement`.

New modules:

- `src/asm/ast.zig` — AST node types (`Span`, `Statement`,
  `Label`, `Unknown`, `Program`).
- `src/asm/parser.zig` — hand-rolled parser walking `[]Token`
  from the include resolver. Multi-error recovery: unknown
  statements emit a diagnostic + `unknown` AST node and the
  parser skips to the next newline, so a single run drains
  every problem at once.

Public API re-exported via `gero.asm_`:

- `Span`, `Statement`, `Label`, `Unknown`, `Program`
- `ParseTree`, `parse(allocator, []Token) → ParseTree`

13 new tests cover: empty program, blank-line / comment-only
input, single label, label-without-newline, label stacks, two
labels on one line, unknown-statement recovery, no-poison after
recovery, multi-error in one run, label span boundaries, and
`file_id` preservation through the parser.
