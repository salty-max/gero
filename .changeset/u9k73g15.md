Unified knit parser byte → AST, plus the source-level include
rework that made it possible.

This is a re-architecture of the asm pipeline, fixing an
inconsistency where knit was used at the lexer level but the
parser was hand-rolled (knit's `ParseState` is byte-only and
the previous token-level include resolution forced the parser
to consume `[]Token`, not bytes). Now the whole asm front-end
is a single knit chain.

**Changes:**

- `resolveIncludes` outputs a fused source string + `SourceMap`
  (instead of a token stream). The parser consumes raw bytes.
- `Token.file_id` removed — the `SourceMap` owns file identity.
- The 30 phase-1 token parsers (`hexP`, `identP`, `colonP`, …)
  are now `pub` and serve as leaf parsers in the new statement
  grammar.
- New `parser.zig` builds statement-level parsers as knit
  combinator chains: `labelP = sequenceOf({identP, blanksP,
  colonP}).map(...)`. Multi-error recovery via `unknown`
  catch-all that always succeeds.
- New `ast.zig`: `Span { start, end }` (no `file_id`),
  `Statement` union, `Label`, `Unknown`, `Program`.

**Scope of this slice (#33 / #80):**

Labels only. Directives + instructions land in follow-up PRs,
each adding one variant to `ast.Statement` and one branch to
the `statementP` choice.

**Test coverage:**

- 12 include tests rewritten for source-level (happy path,
  textual splice, dedup-not-applicable, cycle, depth, missing,
  SourceMap lookup, formatDiagnostic).
- 11 parser tests (empty, labels, multi-label stacks, recovery
  loop, end-to-end with include resolver).
- 4 AST Span tests.
- 59 lexer tests untouched (Token shape change is internal-only).
