**`gero test [pattern]` — asm-level test harness**

New CLI subcommand that walks `tests/asm/` for `.gas` programs
paired with sibling `<name>.expected` files, assembles + boots each
in a fresh VM, captures stdout via the host `int $10` syscall, and
diffs against the golden. Closes #46.

Output mirrors the Cargo-style format from `gero asm`: a
`running N tests` header, one `test <name> ... ok|FAIL` line per
program, a failure section with `expected:` / `got:` blocks, and a
final `N passed[, M failed] (T ms)` summary. `--verbose` adds the
per-test duration to each status line. The optional positional
`pattern` is a substring filter on the test name (the `.gas`
basename without extension).

Exit codes per cli.md §3.4: `0` on all-pass, `7` on any failure,
`1` on host-IO trouble, `2` on bad usage. Tests are capped at a
fixed cycle budget so a buggy program can't hang the runner; the
SRAM-save syscall (`int $21`) is silenced so only stdout is part
of the golden compare.

Ships 4 inaugural `.gas` programs under `tests/asm/programs/`:
`hello.gas` (indexed-load + print loop), `arith.gas` (reg/imm
arithmetic + digit-to-ASCII idiom), `loop.gas` (inc/cmp/jne
counter), and `cmp_branch.gas` (jeq with both arms exercised).

The cli.md §3.4 spec is updated to reflect that v0.1 ships an
asm-level adaptation of the runner; the lang-level (`@test`,
structured assertion diagnostics) form lands with the language
compiler in v0.2.
