Three example `.gas` programs land under `examples/asm/` with their
golden output files — `hello`, `counter`, `fib`.

- `hello.gas` walks a NUL-terminated `data8` string with `mov8`
  indexed-load and prints each byte through `int $10`.
- `counter.gas` prints the 16 hex digits `0..F` via an `inc`/`cmp`/
  `jne` loop.
- `fib.gas` computes fib(10) = 55 = `$37` recursively and prints it
  as 2 hex digits — exercises `call`/`ret`, push/pop discipline, and
  a small hex-digit conversion sub-routine.

Each ships a `<name>.expected` file the test-runner (#48) will diff
against. `examples/**/*.gx` and `*.sav` are gitignored.

The remaining two examples from #47 (`banks.gas`, `save.gas`) are
deferred: the assembler currently hardcodes `bank_count = 0` and
`sram_bank_count = 0` in the `.gx` header (codegen.zig:786-787), so
they need asm-side feature work first.

Discovered while writing `fib.gas`: `mov reg, reg` and `add reg, reg`
use Intel order (dst first) while all other forms use AT&T order
(src first). Filed as #94 — examples document the rule until that's
resolved.
